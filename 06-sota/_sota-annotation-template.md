# SoTA Annotation Template: статус актуальности и пересмотра

**Назначение:** Фиксировать актуальность утверждений, критерии пересмотра и границы применимости для Kids Learning Pack.  
**Читай это, если:** Нужна управляемая SoTA-рамка (SPF/FPF стиль), а не обзор мнений.

---

## 0) Что SoTA здесь означает

`SoTA` в этом pack = **статус актуальности утверждений** + **правила их пересмотра**.

Это **не**:
- литературный обзор "кто прав";
- маркетинговое позиционирование;
- статус зрелости внедрения проекта.

Статус зрелости (`MVP`, `pilot`, `scaled`) ведётся отдельно и не подменяет SoTA.

## 2) Словарь статусов SoTA

| Статус | Когда ставим | Что это значит для решений |
|---|---|---|
| `current` | Есть свежие данные и критерии пересмотра | Можно использовать как рабочую опору |
| `preliminary` | Данные ограничены (пилот/малое n) | Используем с оговорками, пересматриваем чаще |
| `hypothesis` | Эмпирика пока слабая или отсутствует | Тестируем как гипотезу, не как правило |
| `outdated` | Есть новые данные, противоречащие утверждению | Не применять без пересборки |
| `n/a` | Для данного claim SoTA-оценка сейчас не требуется | Не влияет на текущие решения |

## 6) Ритм пересмотра SoTA

- `weekly`: обновляем только источники данных и сигналы рисков.
- `per-cycle`: пересматриваем claims по завершению цикла принципа.
- `quarterly`: ревизия статусов, формул и revision criteria.
- `event-driven`: внеплановый пересмотр при сильном расхождении с практикой.
